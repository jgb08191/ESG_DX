# ESGFit — 기업 맞춤형 ESG 진단 플랫폼

## 📋 프로젝트 개요

**ESGFit**은 기업의 ESG(Environmental, Social, Governance) 경영 전환을 지원하는 **AI 기반 자가진단 플랫폼**입니다.

2026년 자산 2조원 이상 상장사의 ESG 공시 의무화에 대응하기 위해, 기업은 현재의 ESG 성숙도를 파악하고 체계적인 개선 전략이 필요합니다. ESGFit은 **K-ESG 가이드라인, ISO 26000, 중소벤처기업부 가이드라인을 통합**한 61개 기본진단항목으로 기업의 ESG 현주소를 정밀하게 진단하고, 데이터 기반의 개선 방향을 제시합니다.

- **백엔드**: Node.js + Express 서버, `server.js`에서 `0.0.0.0` 바인딩으로 외부 접속 허용
- **보안**: `helmet` 기본 보안 헤더 적용 및 `cors`로 도메인 허용
- **서버리스 준비**: `app` 객체를 `module.exports`로 내보내 Netlify Functions 전환이 용이하도록 구성

### 핵심 특징
- **P·E·S·G 4개 영역**: 정보공시(5) · 환경(17) · 사회(22) · 지배구조(17)
- **61개 진단항목**: 국제 표준 및 국내 규제 통합
- **5단계 성숙도 평가**: 각 문항마다 기업의 현황 수준을 1~5단계로 평가
- **무료 기본 진단**: MVP 단계에서는 모든 기본 진단이 무료
- **규칙 기반 챗봇**: ESG 개념 설명 및 진단 가이드 제공

### **⚙️ JavaScript 파일**

#### `js/main.js` (공유 로직 및 네비게이션)
- **주요 기능**:
  - `navHTML()`: 모든 페이지에서 사용되는 공통 네비게이션 생성
  - `footerHTML()`: 공통 푸터 렌더링
  - `isDiagnoseComplete()`: 진단 완료 여부 확인 (localStorage 기반)
  - `ESGFIT_TOTAL_QUESTIONS`: 총 61개 문항 상수 정의
  - 인증 상태 관리 (로그인/로그아웃)

#### `js/diagnose.js` (ESG 진단 체크리스트 및 채점)
- **주요 기능**:
  - **QUESTIONS 객체**: 61개 진단 문항 데이터 저장
    - `P(정보공시)`: 5개 문항
    - `E(환경)`: 17개 문항 (온실가스, 에너지, 용수, 폐기물 등)
    - `S(사회)`: 22개 문항 (노동, 다양성, 안전보건, 지역사회 등)
    - `G(지배구조)`: 17개 문항 (이사회, 주주권리, 윤리경영 등)
  - 각 문항 구조:
    - `code`: 문항 코드 (예: P-1-1)
    - `cat`: 세부 카테고리
    - `name`: 문항명
    - `q`: 진단 질문문
  - 점수 계산 로직 (1~5 응답값 반영)
  - 영역별 점수 집계

#### `js/chatbot.js` (규칙 기반 ESG 챗봇)
- **주요 기능**:
  - **ESG_KB**: ESG 개념 설명 지식베이스
  - `matchReply(text)`: 사용자 입력 키워드와 매칭하여 응답 생성
  - 챗봇 UI 렌더링 및 메시지 처리
  - 질문 예: "ESG란?", "E영역은?", "점수 기준?", "가입 방법?" 등

## 🚀 주요 데이터 흐름

1. **진단 시작** (`diagnose.html`)
   - 사용자가 P/E/S/G 영역별 탭에서 61개 문항 확인
   - 각 문항에 1~5 응답

2. **데이터 저장** (`localStorage`)
   - 응답값이 자동으로 localStorage에 저장
   ```json
   {
     "esgfit_answers": {
       "P": { "P-1-1": 4, "P-1-2": 3, ... },
       "E": { "E-1-1": 2, ... },
       "S": { ... },
       "G": { ... }
     }
   }
   ```

3. **점수 계산** (`results.html`)
   - localStorage에서 응답값 불러오기
   - 영역별 평균 점수 계산
   - 종합 점수 산출 및 등급 부여
   - 그래프/차트로 시각화

4. **챗봇 지원** (전 페이지)
   - 사용자가 ESG 개념이나 진단 방법 관련 질문
   - 규칙 기반 매칭으로 풀이 제공

## 📊 진단 항목 구성

### P (정보공시) - 5개 문항
- ESG 정보공시 형식 (3개)
- ESG 정보공시 내용 (1개)
- ESG 정보공시 검증 (1개)

### E (환경) - 17개 문항
- 환경경영 목표 및 추진체계 (2개)
- 온실가스 (Scope 1/2/3) 및 감축 (3개)
- 에너지 관리 (2개)
- 용수·폐기물·오염물질 (3개)
- 원재료·유해물질·생물다양성 (4개)

### S (사회) - 22개 문항
- 노동 관행 및 일자리 (4개)
- 직원 다양성·평등 (3개)
- 산업 안전·보건 (3개)
- 맞춤형 교육·기술 (2개)
- 공급망 관리 (3개)
- 지역사회 기여 (4개)

### G (지배구조) - 17개 문항
- 이사회 구성·운영 (3개)
- 주주권리 보호 (2개)
- 감시·감수 기능 (2개)
- 윤리경영·준법 (3개)
- 위험관리 (3개)
- 경영진 보상체계 (2개)

## 🚀 실행 방법

### 1. 필수 환경 설정
1. 프로젝트 루트에 `.env` 파일을 생성합니다.
2. 다음 항목을 설정하세요:
```bash
GOOGLE_API_KEY=your_actual_api_key_here
```
3. 이 프로젝트는 Node.js + Express 백엔드를 사용하며, Gemini 2.5 Flash-Lite API를 호출하기 위한 Google API 키가 필요합니다.

### 2. Node.js 패키지 설치
```bash
npm install
```

### 3. 로컬 서버 실행
```bash
npm start
```
- 서버는 기본적으로 `3000` 포트에서 실행됩니다.
- 브라우저에서 `http://localhost:3000`으로 접속하면 애플리케이션을 확인할 수 있습니다.

### 4. ngrok 터널링 및 외부 도메인 연결
1. ngrok을 사용해 로컬 서버를 외부에 노출합니다:
```bash
ngrok http 3000 --disable-update-check
```
2. ngrok이 실행되면 콘솔에 생성된 `https://...ngrok-free.dev` 주소를 확인합니다.
3. 실제 배포에서는 `www.esgfit.kro.kr` 도메인을 해당 ngrok 주소로 연결했습니다.
   - 이때 `웹 포워딩` 대신 DNS의 `CNAME 별칭` 기능을 사용해 연결한 점이 핵심입니다.
   - `www.esgfit.kro.kr`을 ngrok에서 제공한 `https://...ngrok-free.dev` 주소로 CNAME 설정합니다.

### 5. 주의사항
- ngrok 무료 버전은 매번 실행할 때마다 주소가 변경됩니다. 따라서 ngrok을 재시작하면 새 주소로 DNS `CNAME`을 다시 업데이트해야 합니다.
- 보안 설정을 위해 `server.js`의 CORS 설정은 다음 중 하나여야 합니다:
  - `origin: '*'`로 전체 허용
  - 또는 실행 중인 ngrok 주소(`https://...ngrok-free.dev`)를 특정 허용
- `server.js`가 `0.0.0.0`으로 바인딩되어 있어야 외부에서 접근 가능합니다.
